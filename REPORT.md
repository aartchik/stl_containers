## Инструментарий

Язык: C++23

Компилятор: g++ (ключи: -O2 -Wall -Werror -std=c++23)

Сборка: make

ОС/среда: Linux


## Что реализовано

Реализованы все операции предложенные для выполнения: 4 вида округления и арифметические действия с дробными числами в форматах с плавающей точкой и операции fma и mad.

# Описание

Проект состоит из четырёх файлов исходников и трёх заголовков:

src/

Дальше опишу каждый файл и ключевые функции.

## `main.cpp` — интерфейс и формат вывода

Поддерживаются три формы запуска:

1. `fmt round number`
2. `fmt round op2 a b`  (op2 ∈ {`+`, `-`, `*`, `/`})
3. `fmt round op3 a b c` (op3 ∈ {`fma`, `mad`})

Где `fmt ∈ {s, h}`, `round ∈ {0,1,2,3}`, операнды — шестнадцатеричные слова `0x...` формата.

Основные действия:

* Разбор аргументов и проверка формата входа.
* Вызов соответствующей операции из `operation.cpp`.
* Печать результата через `print_value_hexfloat(fmt, raw)`, которая выводит `value hex` в одной строке.

Вспомогательные функции:

* `parse_hex_u32(const char*)` — перевод строки вида `0x...` в `uint32_t`.
* `parse_i32(const char*)` — чтение режима округления.

## `rounding.hpp/.cpp` — режимы округления и вспомогательные инструменты

Режимы округления задаются числами 0..3. Здесь находятся:

* `int32_t normalize_rounding(int32_t x)`
  Приводит произвольный вход к допустимому диапазону {0,1,2,3}. Используется всеми операциями.

* `int64_t iabs64(int64_t x)`
  Простая абсолютная величина для 64-битных целых.

* `int64_t round_div_to_int(int64_t q, int64_t r, int64_t d, int32_t mode)`
  Унифицированная логика: как корректно «добросить» частное `q` при остатке `r` и делителе `d` под выбранный режим.
  Используется как эталон поведения при нужде рассуждать об округлении в целых.

Окончательное округление чисел с плавающей точкой выполняется в `operation.cpp` при упаковке расширенной мантиссы (см. ниже), там же учитываются `G/R/S`-биты и правило ties-to-even.

## `parse_float.hpp/.cpp` — печать числа в hexfloat

Верхнеуровневые функции:

* `void print_value_hexfloat(char fmt, uint32_t raw)`
* `void print_value_hexfloat_only(char fmt, uint32_t raw)`

Что делается внутри:

* Разбор полей: знак, экспонента, мантисса; распознавание `nan`, `±inf`, `±0`.
* Построение нормализованного представления `S · 2^e2`:

  * Для нормальных чисел мантисса `S` содержит скрытую единицу.
  * Для субнормалов скрытой единицы нет; показатель корректируется.
* Печать в формате `0x1.<дробь>p±k`. Для `half` дробная часть печатается 3 шестнадцатеричными цифрами, для `single` — 6.
* После `value` выводится пробел и сырое `hex` результата.

Вспомогательные рутины:

* `bits_e/bits_f/bias/width_bits/hex_frac_digits(fmt)` — параметры форматов.
* Локальная печать фиксированного числа hex-цифр для `hex`.

## `operation.hpp/.cpp` — арифметика и упаковка/распаковка

### Параметры формата и маски

* `op_bits_e(fmt)`, `op_bits_f(fmt)`, `op_bias(fmt)`, `op_width(fmt)` — разрядности полей и смещение.
* `op_emask(fmt)`, `op_fmask(fmt)` — маски для полей экспоненты и мантиссы.
* `op_pack_raw(fmt, s, e, f)` — сборка слова из полей.
* Готовые конструкторы: `op_make_zero`, `op_make_inf`, `op_make_nan`, `op_make_maxfinite`.

### Распаковка и представление для вычислений

* `void op_unpack(fmt, x, s, z, inf, nan, e, f)` — разбор слова, установка флагов `z/inf/nan`.
* `void op_to_M_e2(fmt, e, f, M, e2)` — получение мантиссы `M` (с учётом скрытой единицы у нормалов) и показателя `e2` так, что число равно `±M · 2^e2`.

### Расширенная мантисса и биты G/R/S

Чтобы не терять точность при выравнивании и уметь честно округлять:

* `uint64_t shift_with_grs(uint64_t m, int32_t d)`
  Сдвиг мантиссы вправо на `d` с формированием `G`/`R`/`S`. Возвращается «расширенная» мантисса: старшие биты — это `m>>d`, младшие три — `GRS`.

* `rshift1_ext(ext)`, `lshift1_ext(ext)`
  Шаговые сдвиги уже в расширенном представлении.

### Нормализация и финальная упаковка

* `uint32_t pack_from_ext(fmt, sign, exp2, ext, mode)`
  Ключевая функция, где происходит:

  1. Приведение вершины мантиссы к диапазону `[1,2)` (или в субнормалы) с коррекцией `exp2`.
  2. Применение режима округления:

     * ties-to-even по `G/R/S` и чётности базы;
     * направленные режимы с учётом знака результата.
  3. Обработка переполнения экспоненты: `inf`/макс. конечное по правилам выбранного режима.
  4. Работа с субнормалами и переносами после округления.
  5. Выбор знака нуля: для `round=3` (`−∞`) — `-0`, иначе `+0`.

* `uint32_t normalize_pack(fmt, sign, mant, exp2, mode)`
  Обёртка: приклеивает `GRS` и вызывает `pack_from_ext`.

## Реализация операций

### Сложение и вычитание

* Ядро: `addsub_core(fmt, a, b, mode, negate_b)`
  Шаги:

  1. Спецслучаи (`nan`, `inf`, нули).
  2. Преобразование к `(M, e2)` для обоих операндов.
  3. Выравнивание по большему показателю: меньшее по масштабу сдвигается через `shift_with_grs`.
  4. Сложение/вычитание расширенных мантисс.
  5. `pack_from_ext` — одна финальная нормализация и округление.

Обёртки: `single_add`, `single_sub`, `half_add`, `half_sub`.

### Умножение

* Ядро: `mul_core(fmt, a, b, mode)`

  1. Спецслучаи (`nan`, `0*inf→nan`, знак как XOR).
  2. `P = Ma * Mb`, показатель `e2a + e2b`.
  3. `normalize_pack` — нормализация с округлением.

Обёртки: `single_mul`, `half_mul`.

### Деление

* Ядро: `div_core(fmt, a, b, mode)`

  1. Спецслучаи (`inf/inf→nan`, `finite/inf→0`, `0/0→nan`, и т.д.).
  2. Нормализованное «длинное деление» с накоплением sticky-бита, чтобы корректно сработало финальное округление.
  3. `normalize_pack`.

Обёртки: `single_div`, `half_div`.

### FMA (слияние умножения и сложения)

* Ядро: `op_fma_core(fmt, a, b, c, mode)`

  1. Спецслучаи IEEE-754: `inf*0→nan`, конфликт знаков бесконечностей, и др.
  2. `P = Ma * Mb`, `e2p = e2a + e2b`.
  3. Выравнивание **в одной шкале**:

     * если `e2p ≥ e2c`: продукт сдвигается на `ff` (для получения «обычной» мантиссы + `GRS`), `c` сдвигается на `e2p − e2c`;
     * если `e2p < e2c`: продукт сдвигается на `ff + (e2c − e2p)`, `c` не трогаю.
  4. Сложение/вычитание расширенных мантисс; один вызов `pack_from_ext`.
     Отличие от `mad` — отсутствует промежуточное округление.

Обёртки: `single_fma`, `half_fma`.

### MAD (умножить, затем сложить)

* Ядро: `op_mad_core(fmt, a, b, c, mode)`

  * Дополнительно учитываю конфликт: если произведение уже бесконечность и `c` тоже бесконечность, возвращается `nan`.
  * Затем просто `mul` → `add` в текущем режиме округления.

Обёртки: `single_mad`, `half_mad`.

## Итоги

1. Реализованы оба формата (half/single) с полной поддержкой особенностей IEEE-754: скрытая единица, субнормалы, `nan/±inf`, знаковый ноль.
2. Округление выполняется строго по выбранному режиму на общем этапе упаковки, для чего используются `G/R/S`-биты.
3. Вывод унифицирован: `hexfloat` + сырой `hex` в одной строке — одинаково для всех операций и форматов.
4. `fma` сделан как «настоящее» слитое вычисление с одним финальным округлением; `mad` — как независимые «mul» и «add` подряд.
